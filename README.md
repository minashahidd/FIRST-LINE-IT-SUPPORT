### **FIRST-LINE-IT-SUPPORT**

### 

This repository documents my hands-on learning and practical work for First-Line IT Support. It showcases technical skills, problem-solving, and professional IT practices across Windows troubleshooting, user accounts, services, Event Viewer, ticket handling, and other core IT support tasks. The structure reflects a real-world approach to first-line IT support, including labs, tickets, cheat sheets, and visual documentation.



##### **Repository Structure:**



##### **Modules**

&nbsp; Each module focuses on a specific area of IT support, such as Windows user issues, system performance, networking basics, Active Directory, software troubleshooting, and ticket management.



######   **Objectives-and-Knowledge**

&nbsp;   Core knowledge and learning objectives for the module, including key tools, commands, and real-world IT practices like escalation, prioritisation, and remote troubleshooting.



###### &nbsp; **Labs**

&nbsp;   Step-by-step documentation of hands-on exercises. Shows the process, actions performed, and outcomes for practical learning.



######   **Ticket-Scenarios**

&nbsp;   Example tickets for common issues with clear documentation and resolutions to demonstrate professional problem-solving.



######   **CheatSheet-and-Summary**

&nbsp;   Quick reference guide for commands, shortcuts, service management, Event Viewer, and communication phrases. Summarises key takeaways for fast recall.



######   **Screenshots**

&nbsp;   Visual evidence of all labs and ticket scenarios. Screenshots are named to match the relevant module and action (e.g., TaskManager\_CPU.png, EventViewer\_LoginFail.png, Ticket\_LoginIssue.png).



##### **Key Highlights:**

1. Demonstrates practical application of core IT support tools
2. Documents troubleshooting, ticketing, and escalation skills
3. Structured, professional format 
4. Includes visual evidence to show hands-on experience
5. Designed for real-world first-line IT support readiness



##### **Usage:**

Review objectives for each module, follow labs for hands-on practice, reference tickets for documentation style, and use cheat sheets for quick recall. Screenshots provide visual confirmation of skills and actions.
